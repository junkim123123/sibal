# 우선순위 화면 흐름 목업 (디자이너용)

아래는 엔지니어가 바로 구현할 수 있고, 디자이너가 바로 시각화해서 프로토타입으로 만들 수 있도록 정리한 화면 흐름 목업이다.

목표는 19단계를 5개 섹션으로 묶어 하이브리드 채팅·폼 경험을 제공하는 것. 각 화면은 데스크탑/모바일 모두 고려된 컴포넌트·카피·상태를 포함한다.

---

## 요약 흐름 (섹션 단위)

1. 시작 화면 (Welcome / 선택)
2. 섹션 A: Product Basics (제품 기본)
3. 섹션 B: Packaging & Logistics (포장·물류)
4. 섹션 C: Compliance & Special Handling (규정·특수요구)
5. 섹션 D: Budget, Schedule, Contact (예산·스케줄·연락처)
6. 리뷰 & 제출 (요약패널 확인 → 제출)

각 섹션은 채팅풍 오프닝 버블 → 카드형 입력(필요 시) → 사용자 응답 버블 → 섹션 완료 메시지 흐름.

---

# 화면별 상세 목업

## 1. Start / Welcome (Zero State)

* 레이아웃

  * 상단: 고정 헤더(로고, 진행바 위치영역)
  * 중앙: 채팅 컨테이너(폭: 데스크탑 700–900px, 모바일 화면 폭)
  * 하단: 고정 전체너비 CTA(다음 또는 시작)
  * 우측(데스크탑): 접이식 요약 패널(초기엔 비어있음)

* 컴포넌트

  * Welcome bubble (봇)

    * 카피: "안녕하세요! 제품 소싱 브리프를 도와드릴게요. 보통 3–4분 소요됩니다. 시작할까요?"
    * 서브: 시간 예측 칩 "약 3분"
    * 액션 칩: [새 소싱 요청] [배송비 확인] [규정 체크]
  * CTA: "시작하기" (Primary, full width on mobile)

* 상태/인터랙션

  * 선택지 탭 또는 시작 클릭 시 첫 질문으로 넘어감
  * 접근성: 환영 텍스트에 aria-live="polite"

---

## 2. 섹션 A: Product Basics (3–5 질문)

목표: 낮은 마찰 질문으로 사용자 참여 확보

### A1. 질문: 어떤 제품인가요?

* UI: Bot bubble -> 입력 카드
* 입력 카드 종류: 텍스트 + LLM 기반 auto-suggest (칩)
* 카피: "제품 이름 또는 링크를 붙여넣어 주세요. (예: '무선 마우스' 또는 Alibaba 링크)"
* 보조: "모름" 버튼 (secondary)
* 인터랙션: 사용자가 입력하면 LLM이 2–3개 표준화 제안(칩) 제공
* 오류: 빈 입력 시 "제품명을 알려주시면 시작할 수 있어요."

### A2. 질문: 수량(의도)

* UI: 슬라이더/칩 카드
* 옵션: 칩들 ["<100", "100–999", "1k–5k", "5k+"], 직접 입력 가능
* 보조: "I don't know" (모름)
* 결과: 요약패널에 수량 반영

### A3. 질문: 이미지/링크 업로드

* UI: 드래그앤드롭 카드 + 카메라 버튼 (모바일)

* 미리보기: 썸네일 + 편집(자르기)/삭제

* 제약: 10MB, JPG/PNG/PDF

* 상태: 업로드 중 스피너, 업로드 성공 토스트

* 섹션 완료 화면: "제품 기본 정보 저장됨. 다음은 포장 및 물류입니다." (타이핑 인디케이터 800ms)

---

## 3. 섹션 B: Packaging & Logistics

목표: 비용 산출에 중요한 물리적/물류 정보 수집

### B1. 포장 형태

* 카드: 라디오/칩 (Retail box / Polybag / Bulk / Custom)
* 서브: "특수 포장(예: 방습, 방충)이 필요한가요?" 체크박스

### B2. 치수/중량

* 카드: 한 번에 입력할 수 있는 '치수 카드' (W × H × D 입력 필드) + 단위 드롭다운 (cm/in)
* 대안: "대략" 토글 → 여러 선택지로 대체
* 유효성: 숫자형만 허용, 단위 표시, 오류 텍스트 (예: '숫자만 입력해주세요')

### B3. 인코텀즈 / 출발지 / 도착지

* 카드: 드롭다운(Incoterms) + 텍스트(도시 또는 우편번호)

* 모름 처리: "모름" 선택 시 추가 질문: "공장 픽업 원하세요?" 예/아니오

* 섹션 완료 메시지: "물류 데이터 저장됨. 다음은 규정 및 특수요구입니다."

---

## 4. 섹션 C: Compliance & Special Handling

목표: 위험·규정 이슈 선탐지

### C1. 위험물 여부

* 카드: 예/아니오 + 상세 (배터리, 액체, 분말 등 체크박스)
* 만약 '예' → 소규모 팝업: "위험물 분류를 위해 추가 정보가 필요합니다. (사람 연결 옵션)"

### C2. 인증 필요 여부

* 카드: 다중선택 칩 (FDA, CE, RoHS, UL, 기타)
* 보조: "모름" → 자동 태그로 검토 요청

### C3. 온도·취급 요구

* 카드: 라디오 (상온 / 냉장 / 냉동 / 민감물품) + 입력란(특수 지시)

* 섹션 완료: "규정 체크 완료. 다음은 예산·연락처입니다."

---

## 5. 섹션 D: Budget, Schedule, Contact

목표: 견적 가능성 판단·연락처 확보

### D1. 예산 범위

* 카드: 칩 (예: "<$5k", "$5k–$20k", "$20k–$50k", "$50k+") + 슬라이더(정밀)
* 보조: "아직 예산 없음" 선택 가능

### D2. 희망 일정

* 카드: 날짜 범위 선택기 (네이티브 모바일 피커 지원)
* 보조: "유연함" 토글

### D3. 연락처 (이메일 우선)

* 입력 카드: 이메일 + 이름(선택) + 회사(선택)
* 타이밍: 이메일은 제출 전/직전에만 필수로 요구 가능
* 접근성: 이메일 필드에 인라인 검증, 오류 문구는 친절하게

### D4. 요약 및 확인 버튼

* 화면: 채팅 내 "요약을 확인하세요" → 우측 요약패널 스냅샷
* CTA: "요청 보내기" (Primary)
* 보조: "나중에 계속하기" (이메일로 매직 링크 전송)

---

## 6. 리뷰 & 제출 후 화면

* 제출 완료 모달: "요청이 접수되었습니다. 보통 24–48시간 내에 스냅샷을 보내드립니다."
* 후속 옵션: "팀과 미팅 예약", "샘플 요청", "프로젝트 저장"
* 트래킹 태그: 제출 이벤트 (analytics)

# 공통 컴포넌트 사양

* 채팅 버블

  * 봇: 색상(brand bg), 폰트 크기 15–16px, 최대 140자, 타이핑 인디케이터
  * 사용자: 반대 색상, 우측 정렬, 복사·수정 가능

* 입력 카드

  * padding 16px, border-radius 12px, elevation subtle
  * 행동 버튼은 반드시 최소 44px 높이, 전체너비 권장(모바일)

* 고정 하단 CTA

  * 데스크탑: 화면 하단에서 떠있는 full-width on mobile, normal width on desktop
  * 상태: disabled / loading / success

* 요약 패널 (데스크탑)

  * width: 280–340px, sticky, 항목 클릭 시 해당 섹션으로 스크롤 점프
  * 모바일: 접이식 서랍(상단에서 슬라이드 업)

* 진행바

  * 상단 고정, 브랜드 색으로 채움, 섹션 라벨 표시

---

# 인터랙션·애니메이션 가이드 (마이크로인터랙션)

* 타이핑 인디케이터: 600–1200ms (무조건 사용하여 응답 지연 느낌 완화)
* 카드 노출: 아래에서 슬라이드 업 (duration 160–200ms) — 레이아웃 쉬프트 최소화
* 업로드 진행: 썸네일에 1px height progress line
* 요약패널 업데이트: 수치 변경 시 작은 pulse(200ms)
* 에러 토스트: 상단 중앙, 3s 자동 사라짐 또는 undo 링크 포함

# 상태·에러 예시 (디자이너 참고)

* 네트워크 장애: "현재 저장이 지연되고 있습니다. 네트워크 연결을 확인하거나 나중에 다시 시도해주세요." + 재시도 버튼
* 유효성 에러: inline — "유효한 이메일 주소를 입력해주세요" (error icon + text)
* 파일 에러: "파일이 너무 큽니다(최대 10MB)."
* 모름 선택: 선택 시 회색 도구팁 "이 경우 추정값으로 계산됩니다. 담당자가 확인할게요."

# 카피 스니펫 (디자이너가 바로 넣을 수 있게)

* Welcome: "안녕하세요! 제품 소싱 브리프를 도와드릴게요. 보통 3–4분 걸립니다. 시작할까요?"
* First question: "어떤 제품을 분석하고 싶으신가요? 이름이나 링크를 붙여넣어 주세요."
* Not sure: "아직 잘 모르겠어요"
* Upload hint: "이미지 또는 상품 페이지 링크를 업로드하세요 (JPG, PNG, PDF, 최대 10MB)."
* Save & resume: "중간에 멈춰야 하나요? 이메일을 남기시면 이어서 할 수 있는 링크를 보내드립니다."

---

# 반응형 / 접근성 노트

* 본문 텍스트 최소 16px (모바일), 데스크탑 15–16px 권장.
* 버튼 터치 타깃 최소 44px.
* 대비: 텍스트 대 배경 4.5:1 이상.
* 키보드 내비게이션: 모든 인터랙션은 Tab 순서에 포함, aria-label 제공.
* 장애 접근: aria-live 지역으로 봇 응답 읽기 지원.

# 기술 스택 참고

* 프레임워크: Next.js App Router (기존 프로젝트 구조 활용)
* 상태 관리: React useState/useReducer 또는 Zustand
* 폼 검증: Zod 또는 React Hook Form
* 파일 업로드: Next.js API Route + 클라우드 스토리지
* 애니메이션: Framer Motion 또는 CSS transitions
* 접근성: ARIA attributes, 키보드 네비게이션

---

# 기존 컴포넌트 재사용 가능 여부

프로젝트에 이미 존재하는 컴포넌트:
- `web/components/chat/*` - 채팅 버블 컴포넌트
- `web/components/ui/card.tsx` - 카드 컴포넌트
- `web/components/ui/button.tsx` - 버튼 컴포넌트
- `web/app/chat/page.tsx` - 온보딩 채팅 인터페이스 (참고용)
- `web/app/analyze/chat/page.tsx` - Dr B 스타일 intake flow (참고용)

재사용 가능한 패턴:
- 채팅 메시지 상태 관리
- 단계별 질문 플로우
- 입력 타입별 UI 렌더링
